Day 1: Base Maps, Creating a Map, Symbology, & Pulling in Data

Learn the Map Viewer, pick a basemap (OpenStreetMap is a great neutral base), set your coordinate system & initial extent, symbolize layers, and add external data.

1) Create your first web map

  1. In ArcGIS Online, click Map to open Map Viewer.
  2. Choose Basemap → OpenStreetMap.
  3. Zoom to your area of interest and click Map Properties and then under Map View toggle Preserve Map Scale
  4. In properties Save the map.

3) Symbolize a layer

  1. Add a layer (search the LLBO DRM or drag a file).
  2. Open Styles → pick a drawing style (e.g., Types (Unique symbols)).
  3. Tune the color, size, and legend for clarity.

4) Pull in data from elsewhere

Add CSV/GeoJSON/Feature layers via URL or by uploading. For open data, look for services that expose a Feature Service URL so updates flow through to your map.

Example: “Pokémon GO” map with an OpenStreetMap basemap

Use OSM as the basemap and overlay your own points (e.g., PokéStops) for a clean, legible look.

Web-app example: Invasion of America

A more built-out ArcGIS web application showing Indigenous land cessions over time. Explore the timeline, layers, and pop-ups for inspiration.

Day 2: Feature Layers, Points & Polygons, Attribute Tables

Create and manage feature layers, work with attribute tables, and practice safe editing workflows.

1) Build a feature layer

  1. From Content → New item → Feature layer.
  2. Pick geometry type: points, lines, or polygons.
  3. Define fields you need (text, number, date, etc.).

2) Edit the attribute table

  1. Open the layer → Data tab.
  2. Add/edit fields, set domains (coded lists), and calculate values.
  3. Use Editor to add or modify features with proper tracking.

3) Points vs. polygons

Use points for discrete locations (addresses, facilities) and polygons for areas (parcels, districts). Pick the geometry that best represents the question you’re answering.

Reference: LLBO DRM

The LLBO DRM is a great resource to see how a layer is used to communicate data. Explore symbology, pop-ups, and metadata in existing items to guide your own designs.

Day 3: Story Maps—Maps With Narrative

Combine maps, images, and text into a compelling story. Great for reports, grant applications, and public engagement.

1) Create a Story Map

  1. From Content → New item → StoryMap.
  2. Pick a Theme that matches your brand.
  3. Add Map blocks and connect to your web maps.

2) Pull in images & media

Use Image or Gallery blocks to add photos and diagrams. Credit your sources and add alt text for accessibility.

3) Complement maps with narrative

Lead with a clear headline, keep paragraphs short, and use callouts to explain what the reader should notice on the map. Link back to data sources and the LLBO DRM where appropriate.
